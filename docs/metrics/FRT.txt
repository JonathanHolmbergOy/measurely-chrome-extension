FRT - Frame Titles
==================

OVERVIEW
--------
Rule ID: frame-title
Impact: Serious
Category: Accessibility (Frame Elements)
WCAG: 2.0 Level A (4.1.2 Name, Role, Value, 2.4.1 Bypass Blocks)

Description:
Ensures iframe and frame elements have title or aria-label attributes. Frame titles help screen reader users understand the purpose and content of embedded content before entering it.

Thresholds:
- Good: 0 violations
- Poor: â‰¥ 1 violation


WHY IT MATTERS
--------------
Frame titles:
- Identify iframe purpose for screen readers
- Enable users to skip irrelevant embedded content
- Provide context before loading frame content
- Meet WCAG 2.4.1 and 4.1.2 requirements


DETECTION METHOD
----------------
Client-side validation that:
1. Finds all <iframe> and <frame> elements
2. Checks for title or aria-label attribute
3. Reports frames without accessible names
4. Includes src and ID for identification


HOW TO FIX
----------
1. Add descriptive title attribute (preferred)
2. Use aria-label for dynamic descriptions
3. Ensure title describes iframe purpose, not just "iframe"
4. Update titles if iframe content changes


EXAMPLE ISSUES
--------------
Bad:
<iframe src="https://example.com/map"></iframe>
<iframe src="https://youtube.com/embed/xyz"></iframe>
<iframe title="iframe"></iframe>

Good:
<iframe src="https://example.com/map" title="Store location map"></iframe>
<iframe src="https://youtube.com/embed/xyz" title="Product demo video"></iframe>
<iframe aria-label="Live chat support" src="/chat"></iframe>


COMMON IFRAME TYPES
--------------------
Maps:
<iframe src="..." title="Our office location on Google Maps"></iframe>

Videos:
<iframe src="..." title="Tutorial: How to install the software"></iframe>

Ads:
<iframe src="..." title="Advertisement"></iframe>

Embedded Content:
<iframe src="..." title="Customer testimonials"></iframe>


BROWSER SUPPORT
---------------

- Screen Readers: Critical for context
