DLS - Definition List Structure
================================

OVERVIEW
--------
Rule ID: definition-list
Impact: Serious
Category: Accessibility (Document Structure)
WCAG: 2.0 Level A (1.3.1 Info and Relationships)

Description:
Ensures dl (definition list) elements only contain dt, dd, div, script, or template elements as direct children. Invalid structure breaks screen reader navigation and semantic understanding.

Thresholds:
- Good: 0 violations
- Poor: â‰¥ 1 violation


WHY IT MATTERS
--------------
Proper definition list structure:
- Enables screen readers to navigate term/definition pairs
- Maintains semantic document structure
- Supports list navigation shortcuts
- Meets WCAG 1.3.1 requirements


DETECTION METHOD
----------------
Client-side validation that:
1. Finds all <dl> elements
2. Checks direct children are only <dt>, <dd>, <div>, <script>, or <template>
3. Reports lists with invalid children
4. Identifies invalid child elements


HOW TO FIX
----------
1. Move non-dt/dd content outside <dl>
2. Wrap groups in <div> if needed
3. Use semantic structure
4. Validate term/definition pairing


EXAMPLE ISSUES
--------------
Bad:
<dl>
    <span>Term 1</span>
    <span>Definition 1</span>
</dl>

<dl>
    <p>Invalid paragraph</p>
    <dt>Term</dt>
    <dd>Definition</dd>
</dl>

Good:
<dl>
    <dt>Term 1</dt>
    <dd>Definition 1</dd>
</dl>

<dl>
    <div>
        <dt>Term 1</dt>
        <dd>Definition 1</dd>
    </div>
    <div>
        <dt>Term 2</dt>
        <dd>Definition 2</dd>
    </div>
</dl>


GLOSSARY PATTERN
----------------

    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    
    <dt>CSS</dt>

MULTIPLE DEFINITIONS
--------------------

    <dt>Coffee</dt>

    <dd>A dark brown color</dd>
    
    <dt>Tea</dt>
    <dd>An aromatic beverage made from Camellia sinensis leaves</dd>
</dl>
