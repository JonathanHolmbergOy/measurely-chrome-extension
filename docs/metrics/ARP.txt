ARP - ARIA Required Parent
===========================

OVERVIEW
--------
Rule ID: aria-required-parent
Impact: Critical
Category: Accessibility (ARIA Validation)
WCAG: 2.0 Level A (1.3.1 Info and Relationships)

Description:
Ensures elements with ARIA roles that require specific parent roles are properly contained by them. Correct parent-child relationships are crucial for screen readers to understand document structure and navigate effectively.

Thresholds:
- Good: 0 violations
- Poor: ≥ 1 violation


WHY IT MATTERS
--------------
Required parent roles ensure:
- Proper context for child elements
- Correct screen reader announcements
- Expected keyboard navigation patterns
- Semantic document structure


REQUIRED PARENTS BY ROLE
-------------------------
- listitem → list
- option → listbox
- menuitem → menu or menubar
- menuitemcheckbox → menu or menubar
- menuitemradio → menu or menubar
- radio → radiogroup
- tab → tablist
- treeitem → tree
- cell/gridcell/columnheader/rowheader → row


DETECTION METHOD
----------------
Client-side validation that:
1. Finds elements with child roles requiring parents
2. Traverses DOM tree to find parent with required role
3. Reports violations if proper parent not found
4. Provides fix with required parent role


HOW TO FIX
----------
1. Wrap child element in parent with required role
2. Ensure parent role is appropriate
3. Use semantic HTML when possible
4. Maintain proper nesting structure


EXAMPLE ISSUES
--------------
Bad:
<div>
    <div role="listitem">Item 1</div>
    <div role="listitem">Item 2</div>
</div>

<div>
    <div role="tab">Tab 1</div>
    <div role="tab">Tab 2</div>
</div>

Good:
<div role="list">
    <div role="listitem">Item 1</div>
    <div role="listitem">Item 2</div>
</div>

<div role="tablist">
    <div role="tab">Tab 1</div>
    <div role="tab">Tab 2</div>
</div>

Best (Semantic HTML):
<ul>
    <li>Item 1</li>
    <li>Item 2</li>
</ul>


BROWSER SUPPORT
---------------
- Chrome/Edge: Full support
- Firefox: Full support
- Safari: Full support
- Screen Readers: Critical for context
