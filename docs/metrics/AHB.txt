AHB - ARIA Hidden Body
======================

OVERVIEW
--------
Rule ID: aria-hidden-body
Impact: Critical
Category: Accessibility (ARIA Validation)
WCAG: 2.0 Level A (1.3.1 Info and Relationships, 4.1.2 Name, Role, Value)

Description:
Detects if the body element has aria-hidden="true", which would hide the entire page from screen readers. This critical error makes the entire website inaccessible to assistive technology users.

Thresholds:
- Good: 0 violations
- Poor: 1 violation


WHY IT MATTERS
--------------
aria-hidden="true" on body:
- Hides all page content from screen readers
- Makes entire website completely inaccessible
- Violates WCAG 1.3.1 and 4.1.2
- No content is announced to assistive technology users


COMMON CAUSES
-------------
1. JavaScript frameworks setting aria-hidden during loading
2. Modal/overlay implementations incorrectly hiding body
3. Copy-paste errors from other elements
4. Testing/debugging code left in production


DETECTION METHOD
----------------
Client-side validation that:
1. Checks body element for aria-hidden attribute
2. Verifies if value is "true"
3. Reports critical violation if found


HOW TO FIX
----------
1. Remove aria-hidden="true" from body element
2. Use aria-hidden only on specific content sections
3. For modals, use inert attribute on background instead
4. Audit JavaScript that manipulates aria-hidden


EXAMPLE ISSUES
--------------
Bad:
<body aria-hidden="true">
    <div>All content is now hidden from screen readers</div>
</body>

Good:
<body>
    <div>Content is accessible</div>
    <div aria-hidden="true">Only this specific div is hidden</div>
</body>

For Modals:
<body>
    <main inert>Main content (background)</main>
    <div role="dialog" aria-modal="true">
        <h2>Modal content</h2>
    </div>
</body>


BROWSER SUPPORT
---------------
- Chrome/Edge: Full support
- Firefox: Full support
- Safari: Full support
- Screen Readers: All affected
