MVS - Meta Viewport Scale
=========================

OVERVIEW
--------
Rule ID: meta-viewport
Impact: Critical
Category: Accessibility (Viewport/Zoom)
WCAG: 2.1 Level AA (1.4.4 Resize Text)

Description:
Detects viewport meta tags that prevent zooming (user-scalable=no or maximum-scale ≤ 1). Preventing zoom violates WCAG 1.4.4 and makes content inaccessible to users who need magnification.

Thresholds:
- Good: 0 violations
- Poor: 1 violation


WHY IT MATTERS
--------------
Allowing zoom is essential for:
- Low vision users who need magnification
- Users with presbyopia (age-related near vision loss)
- Mobile users viewing detailed content
- WCAG 1.4.4 Level AA compliance


DETECTION METHOD
----------------
Client-side validation that:
1. Finds <meta name="viewport"> element
2. Checks for user-scalable=no or maximum-scale ≤ 1
3. Reports viewport preventing zoom
4. Shows current viewport content value


HOW TO FIX
----------
1. Remove user-scalable=no from viewport
2. Remove maximum-scale or set ≥ 2
3. Allow browser default zoom behavior
4. Test site still works when zoomed


EXAMPLE ISSUES
--------------
Bad:
<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no">
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
<meta name="viewport" content="width=device-width, user-scalable=0">

Good:
<meta name="viewport" content="width=device-width, initial-scale=1">
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=5">


RESPONSIVE DESIGN
-----------------
Proper viewport configuration for responsive sites:

<meta name="viewport" content="width=device-width, initial-scale=1">

This allows:
- Content adapts to device width
- Initial zoom level at 1x
- User can pinch-zoom to 500%+ (browser default)
- Meets WCAG zoom requirements


WHY DEVELOPERS DISABLE ZOOM
----------------------------
Common (bad) reasons:
- "Prevents accidental zoom during interaction"
  → Solution: Use CSS touch-action instead
- "Keeps mobile design intact"
  → Solution: Design should work when zoomed
- "Improves performance"
  → Solution: Negligible performance difference


CSS ALTERNATIVE
---------------
Instead of disabling zoom, prevent double-tap zoom on buttons:

button {
    touch-action: manipulation;
}


BROWSER SUPPORT
---------------
- Chrome/Edge: Respects user-scalable setting
- Firefox: Respects user-scalable setting
- Safari iOS: Partially ignores restrictions (good for accessibility)
- Screen Readers: N/A (but magnification users affected)
