AAT - Area Alt Text
===================

OVERVIEW
--------
Rule ID: area-alt
Impact: Critical
Category: Accessibility (Image Maps)
WCAG: 2.0 Level A (1.1.1 Non-text Content, 2.4.4 Link Purpose)

Description:
Ensures <area> elements in image maps have alt text. Image map areas define clickable regions within images, and without alt text, screen reader users cannot identify or activate these interactive regions.

Thresholds:
- Good: 0 violations
- Poor: â‰¥ 1 violation


WHY IT MATTERS
--------------
Area alt text:
- Describes clickable regions in image maps
- Enables keyboard navigation of map areas
- Provides link context for screen readers
- Meets WCAG 1.1.1 and 2.4.4 requirements


DETECTION METHOD
----------------
Client-side validation that:
1. Finds all <area> elements in <map> tags
2. Checks for alt attribute presence and content
3. Reports areas without alt text
4. Includes href and coords for identification


HOW TO FIX
----------
1. Add alt attribute to every <area> element
2. Describe the destination or purpose
3. Keep descriptions concise and meaningful
4. Don't include "link" or "image" in alt text


EXAMPLE ISSUES
--------------
Bad:
<map name="nav">
    <area shape="rect" coords="0,0,100,50" href="home.html">
    <area shape="rect" coords="100,0,200,50" href="about.html" alt="">
</map>

Good:
<map name="nav">
    <area shape="rect" coords="0,0,100,50" href="home.html" alt="Home">
    <area shape="rect" coords="100,0,200,50" href="about.html" alt="About Us">
    <area shape="rect" coords="200,0,300,50" href="contact.html" alt="Contact">
</map>
