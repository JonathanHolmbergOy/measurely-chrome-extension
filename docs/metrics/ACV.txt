ACV - Autocomplete Valid
========================

OVERVIEW
--------
Rule ID: autocomplete-valid
Impact: Serious
Category: Accessibility (Forms)
WCAG: 2.1 Level AA (1.3.5 Identify Input Purpose)

Description:
Ensures form input autocomplete attributes use valid values from the HTML specification. Invalid autocomplete values prevent browsers from autofilling forms and violate WCAG 1.3.5.

Thresholds:
- Good: 0 violations
- Poor: â‰¥ 1 violation


WHY IT MATTERS
--------------
Valid autocomplete attributes:
- Enable browser autofill for forms
- Help users complete forms faster
- Support users with cognitive disabilities
- Meet WCAG 1.3.5 Level AA requirements


VALID AUTOCOMPLETE VALUES
--------------------------
Personal: name, given-name, family-name, additional-name, nickname
Email/Tel: email, tel, tel-country-code, tel-national, tel-area-code
Address: street-address, address-line1-3, country, postal-code, city
Payment: cc-name, cc-number, cc-exp, cc-csc, cc-type
Other: username, new-password, current-password, one-time-code, organization, bday


DETECTION METHOD
----------------
Client-side validation that:
1. Finds inputs with autocomplete attribute
2. Validates against HTML autocomplete spec
3. Reports invalid values
4. Suggests correct alternatives


HOW TO FIX
----------
1. Use valid autocomplete values from spec
2. Match input purpose to autocomplete token
3. Remove invalid autocomplete attributes
4. Test autofill functionality


EXAMPLE ISSUES
--------------
Bad:
<input type="text" autocomplete="fullname">
<input type="email" autocomplete="email-address">
<input type="tel" autocomplete="phone">

Good:
<input type="text" autocomplete="name">
<input type="email" autocomplete="email">
<input type="tel" autocomplete="tel">


COMMON FORM PATTERNS
---------------------
Name Fields:
<input type="text" autocomplete="given-name" placeholder="First name">
<input type="text" autocomplete="family-name" placeholder="Last name">

Address Form:
<input autocomplete="street-address">
<input autocomplete="address-level2" placeholder="City">
<input autocomplete="postal-code">
<input autocomplete="country-name">

Login Form:
<input type="email" autocomplete="username">
<input type="password" autocomplete="current-password">

Credit Card:
<input autocomplete="cc-name">
<input autocomplete="cc-number">
<input autocomplete="cc-exp">
<input autocomplete="cc-csc">


BROWSER SUPPORT
---------------
- Chrome/Edge: Full autofill support
- Firefox: Full autofill support
- Safari: Full autofill support
- Screen Readers: Announces input purpose
